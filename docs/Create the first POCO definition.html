For the examples I use the Microsoft sample database Adventure Works.

Start PocoGen from the start menu (or start PocoGen.GUI.exe). You will see the main screen:

Select “Microsoft SQL Server” as database type and enter the connection string to your database. Notice that the “Connection string documentation” link is enabled once you select a database type. It will bring you to a description of the connection string options.
For my local Adventure Works database, the connection string is: Data Source=(local);Initial Catalog=AdventureWorks2012;Integrated Security=SSPI

Click on the “Test connection” button. A message box should appear which shows “Connection succeeded.”. Otherwise you have an error in your connection string, the database server is unavailable, your credentials are wrong or you don’t have access.

Now click on “Load Tables” and select the tab Output configuration. Select “POCOs” in the list of available writers and add it with the button. It should look like this:

By clicking on “Preview” in the blue box you can preview the result of the generated code:

That’s it, you created your first POCO definition with PocoGen! You can save your Poco-project in the main menu: Project > Save definition as… Please note that the connection string will be saved in the file in clear text. That means that anyone who can access the file with a text editor can see the credentials for the database if you needed to specify them in the connection string.
