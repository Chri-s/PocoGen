Connect to your database (explained in Create the first POCO definition) and click on “Load tables” on the tab “Connection”. Go to the tab “Tables” and it should look like this:

You see one blue box for each table or view. The button opens the list of columns, the next column marks the type of object:  marks a table and a view. The checkbox shows whether the table will be included in the output. You can uncheck the checkbox if you want to ignore a table or view. The bold text is the name of the generated class and the parenthesized text shows the name of the table or view.

 

To change the name of a class (or column) click on the name. A textbox appears and you can change the name. Just press enter when you are done:

If you want to make an identical change to many tables, it is too cumbersome to do it by hand. Suppose you want to replace the word “Product” with “Item” in the class names. Go to the tab “Table Naming” and select “Replace” in the combo box. Click the button. Click on the settings button () of the replace name generator and set these changes:

Close the settings window and go back to the tab “Tables”. Scroll down until you see the tables below PhoneNumberType. You see that the class names starts with Item while the table names start with Product:

Every time the tables are loaded, the table and column name generators are applied and after that your custom name changes are applied. For example, if you add a table named ProductStorageLocation, the POCO class is automatically named ItemStorageLocation.

There are other table and column name generators. Look at the combo boxes in the tabs and check whether they fit your need.
