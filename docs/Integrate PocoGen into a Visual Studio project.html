You can integrate PocoGen into Visual Studio like a T4 template so that the database will be queried and the output files recreated every time you build your project. Because PocoGen runs as a pre-build task, it can be used in every Visual Studio Version and will also run when you invoke MSBuild.

Therefore you have to open the project properties, select the tab “Build Events” and enter this line into the upper text box (“Pre-build event command line:”):

Start /B PocoGen.Runner.exe "$(ProjectDir)YourPocoFile.poco"

This assumes that you installed PocoGen with the Setup and included the Command Line Generator. Otherwise you have to specify the complete path to PocoGen.Runner.exe. If your .poco file is not in the same directory as the project file (.csproj/.vbproj) you have to change the path in quotes to the relative path.
